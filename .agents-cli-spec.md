# Home Services Intake Showcase Agent

## Overview

This agent is a public showcase built by Criterix to demonstrate a production-shaped
agent pattern: an ADK 2.0 graph workflow that triages inbound home-services leads,
classifies intent, urgency, and trade, and produces a clean, structured handoff for a
human dispatcher or CSR.

It is a deliberately scoped, safe subset of the larger "Home Services Revenue Desk"
concept. It uses mock, in-memory tools only — there are no real CRM, phone, or
messaging integrations — so it is safe to run and share publicly without any customer
data.

The graph controls the critical path. Language models are used only where natural
language understanding genuinely helps (classification and reply drafting); routing
and safety decisions are made by deterministic code nodes.

```
START
  -> normalize_lead        (clean the inbound message)
  -> classifier_agent      (LLM: intent, urgency, trade, missing fields)
  -> policy_gate           (deterministic routing + safety override)
  -> book | estimate | human   (LLM: draft a helpful reply for the chosen path)
  -> format_handoff        (mock service-area lookup + mock lead save -> structured handoff)
```

## Example Use Cases

### Inbound lead qualification (book path)
Input: "My upstairs AC is blowing warm air. We're in Plano. Can someone come tomorrow morning?"
Expected: classified as `book_service`, urgency `urgent`, trade `hvac`, service city `Plano`
(in service area). A booking-prep reply is drafted and a structured handoff is produced.
The agent never confirms an appointment time.

### Estimate follow-up (estimate path)
Input: "I got a quote last week to replace my condenser and I'm still deciding."
Expected: classified as `estimate_follow_up`. A polite follow-up reply is drafted that
answers only general questions and escalates any price-change or discount request to a
human.

### High-risk escalation (human path)
Input: "I want a refund for the service you did yesterday."
Expected: `policy_gate` forces a human handoff regardless of the model's classification,
because billing, refunds, cancellations, and complaints are never handled autonomously.

### Missing information
Input: "My heater isn't working, can someone help?"
Expected: classifier flags `missing_fields` (for example service address / city / phone).
The handoff records what is still needed before a human follows up.

## Tools Required

All tools in this showcase are mock and in-memory. They illustrate where real
integrations would attach in a client engagement.

- `lookup_service_area(city)` — checks a city against a static DFW-area list and returns
  `in_area`, `out_of_area`, or `unknown`. Stands in for a real service-area / dispatch
  boundary check.
- `save_lead_draft(handoff)` — simulates writing a structured lead/handoff record to a
  CRM and returns a generated draft id. Stands in for a ServiceTitan / Housecall Pro /
  Jobber write.

## Constraints & Safety Rules

The agent must NOT:
- quote binding prices or confirm appointment times
- process refunds, discounts, cancellations, or reschedules
- handle complaints, legal, permit, insurance, or liability questions autonomously
- fabricate service history, warranty status, or financing terms

The agent MUST:
- route billing, refunds, cancellations, complaints, and any `requires_human` case to a
  human handoff, regardless of the model's own classification
- record missing intake fields rather than inventing them
- keep all customer data inside the mock tools (nothing leaves the demo boundary)

## Success Criteria

- Route accuracy: book vs. estimate vs. human handoff matches the intent.
- Escalation correctness: billing / refund / cancellation / complaint inputs always reach
  the human path.
- Handoff completeness: structured output captures intent, urgency, trade, service-area
  status, missing fields, recommended route, and the drafted reply.
- Safety: no binding price or confirmed appointment time appears in any reply.

## Deployment

- Local prototyping with `agents-cli` and ADK 2.0.
- Production host: Gemini Enterprise Agent Platform Agent Runtime (sessions managed by
  the platform).

## Out of Scope (future Revenue Desk phases)

Real CRM/phone/SMS integrations, Memory Bank cross-session personalization, RAG over SOPs
and financing FAQs, and human-in-the-loop approval pauses are intentionally excluded from
this v1 showcase.
